# 前后端对接清单

更新时间：2026-07-01 约20:30 CST

本文给后端同学快速对齐：React 前端已先做教师端主流程和部分前端先行入口；下面列出各后端分工上线时，前端需要配合什么。

| 分工 | 后端要交付 | 前端现在已做 | 前端还没做 / 需配合 |
|---|---|---|---|
| OCR / 文件摄入 | `OCRProvider` 抽象；图片、扫描 PDF、手写作业转文本/LaTeX；错误可人工订正 | 上传页已支持题目/作答文本类流程、识别结果预览和人工编辑 | 后端接口稳定后，放开图片类型、展示 OCR 进度/置信度/失败原因；更新上传页文案，不再说 OCR 待接入 |
| Agent 算法 / 批改流程 | 更稳定的抽题、解析、批改、复核信号、自动反馈语言；保持 progress events | 教师 Setup、上传校对、启动批改、进度轮询、结果总览/按题/学生/题目详情已接入 | 若返回字段变化，前端要同步类型、复核标签、结果卡片、图表和 NL 分析；批改语言真正自动识别后，移除兼容常量说明 |
| RAG / 知识库 | user-scoped KB 模型；上传/列表/删除 API；任务引用个人 KB；批改时合并 task/user 检索 | task-scoped KB 已真实接入；新增 `/knowledge-base` 前端先行页、Setup 选择器、“上传本任务资料时加入个人清单” | 后端 API 完成后，把 localStorage 清单替换为真实 API；Setup 选择结果写入任务配置；去掉“前端先行/不参与批改”免责声明 |
| 数据库 / 对象存储 | PostgreSQL 持久化用户/任务/结果/BYOK/KB；对象存储保存原文件；权限隔离 | 前端按 `/tasks/*`、`/analytics/*` 真实 API 工作；已有 owner-scoped 体验 | 持久化上线后，前端需处理分页、加载更多、长期历史任务、删除确认和“数据会丢”类文案下线 |
| BYOK / 共享池安全 | per-user 额度、共享池熔断、单专家/单采样强制、审计日志 | BYOK Experts 页面可增删启停 key；前端不暴露共享池管理入口 | 后端给出额度/错误码后，前端展示剩余额度、共享池不可用原因、引导用户改用 BYOK |
| LMS / 课程作业 | LTI/SSO、课程映射、作业映射、提交读取、成绩回传、角色权限 | React 当前不展示 LMS/课程/作业入口，仅保留隐藏类型 | 后端和学校测试环境就绪后，再开放课程/作业/LMS 导航和页面；同时调整教师主流程入口 |

## 需要一起改的触发点

- **接口字段变了**：后端先给 response 示例或 OpenAPI，前端同步 `src/types/*` 和页面展示。
- **能力从“待接入”变“已可用”**：前后端一起删免责声明，避免用户看到过时文案。
- **新增长期数据能力**：数据库/对象存储上线后，前端要补分页、空态、删除/恢复语义。
- **新增用户可见入口**：LMS、全局 KB、OCR 图片上传这类入口必须等后端可跑通再正式开放。
